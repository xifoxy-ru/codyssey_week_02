# 미션: Git과 함께하는 Python 첫 발자국

## 1. 프로젝트 개요
이 프로젝트는 Python으로 만든 콘솔 기반 퀴즈 게임입니다.  
사용자는 메뉴를 통해 퀴즈를 풀고, 새로운 퀴즈를 추가하고, 등록된 퀴즈 목록을 확인하고, 최고 점수와 최근 점수 기록을 확인할 수 있습니다.  
또한 프로그램을 종료한 뒤 다시 실행해도 퀴즈 데이터와 점수 기록이 유지되도록 `state.json` 파일을 사용해 데이터를 저장합니다.

이 프로젝트는 단순한 문법 연습보다 **객체지향 구조 설계**를 중심에 두고 진행했습니다.  
문제 데이터를 하나의 객체로 다루고, 게임 전체 흐름을 별도 객체로 관리하는 구조를 통해 클래스와 역할 분리를 연습하는 것을 목표로 했습니다.

추가로 보너스 과제도 함께 반영하여, 기본 기능 외에 문제 순서 랜덤 출제, 문제 수 선택, 힌트 기능, 퀴즈 삭제 기능, 점수 기록 히스토리 저장까지 구현했습니다.

---

## 2. 퀴즈 주제 선정 이유
이 프로젝트의 퀴즈 주제는 **생물의 분류와 특징 퀴즈**입니다.

퀴즈 주제는 단순히 문제를 만들기 쉬운 분야보다, 객체의 속성과 역할을 설계하기 좋은 방향으로 정하고 싶었습니다.  
생물 주제는 동물, 인간, 서식 환경, 분류 체계, 특징 등 다양한 정보를 문제 데이터에 반영할 수 있어 객체지향 구조를 연습하기에 적합하다고 보았습니다.

또한 문제마다 질문, 선택지, 정답, 힌트 같은 속성을 자연스럽게 둘 수 있어 `Quiz` 객체를 설계하기에 알맞다고 판단했습니다.

## 5. 프로그램 동작 방식

### 5-1. 퀴즈 풀기
퀴즈를 시작하면 먼저 몇 문제를 풀지 선택할 수 있습니다.  
이후 등록된 퀴즈가 랜덤 순서로 출제되며, 각 문제마다 4개의 선택지가 함께 출력됩니다.  
사용자는 정답 번호를 입력해 문제를 풀 수 있습니다.

또한 각 문제에서 힌트를 볼 수 있으며, 힌트를 사용할 경우 점수가 차감됩니다.  
모든 문제를 푼 뒤에는 정답 수, 힌트 사용 횟수, 최종 점수, 최고 점수가 출력됩니다.

### 5-2. 퀴즈 추가
사용자가 직접 문제, 선택지 4개, 정답 번호, 힌트를 입력해 새 퀴즈를 추가할 수 있습니다.  
추가된 퀴즈는 메모리뿐 아니라 `state.json`에도 반영됩니다.

### 5-3. 퀴즈 목록
현재 저장된 퀴즈 목록을 확인할 수 있습니다.  
문제 번호와 문제 내용이 출력되며, 퀴즈가 하나도 없는 경우 안내 메시지를 출력합니다.

### 5-4. 점수 확인
현재 최고 점수와 최근 점수 기록을 확인할 수 있습니다.  
점수 기록에는 플레이 시각, 푼 문제 수, 맞힌 문제 수, 힌트 사용 횟수, 최종 점수가 저장됩니다.

### 5-5. 퀴즈 삭제
등록된 퀴즈 중 원하는 번호를 선택해 삭제할 수 있습니다.  
삭제 전에 확인 질문을 한 번 더 받고, 삭제가 완료되면 `state.json`에도 반영됩니다.

## 6. 코드 구조
이 프로젝트는 최소 2개 이상의 클래스를 사용해 역할을 분리하여 구현합니다.

### `Quiz`
개별 퀴즈 1개를 표현하는 클래스입니다.

#### 주요 속성
- `question`: 문제 내용
- `choices`: 선택지 4개
- `answer`: 정답 번호
- `hint`: 힌트 내용

#### 주요 역할
- 문제 정보 보관
- 문제 출력
- 정답 확인
- 저장용 딕셔너리 변환

### `QuizGame`
게임 전체 흐름을 관리하는 클래스입니다.

#### 주요 속성
- `quizzes`: 전체 퀴즈 목록
- `best_score`: 최고 점수
- `score_history`: 점수 기록 히스토리
- `default_quiz_file`: 기본 퀴즈 데이터 파일 경로
- `state_file`: 상태 저장 파일 경로

#### 주요 역할
- 메뉴 출력
- 사용자 입력 처리
- 퀴즈 진행
- 퀴즈 추가
- 퀴즈 목록 출력
- 점수 확인
- 퀴즈 삭제
- 파일 저장/불러오기
- 예외 처리 및 종료 처리

## 8. 데이터 파일 설명

### 8-1. `default_quizzes.json`
초기 기본 퀴즈 데이터를 보관하는 파일입니다.  
프로그램 최초 실행 시 `state.json`이 없으면 이 파일을 기준으로 상태 파일을 생성합니다.

예시 구조:

```json
[
  {
    "question": "박쥐는 어떤 분류에 속할까요?",
    "choices": ["조류", "파충류", "포유류", "양서류"],
    "answer": 3,
    "hint": "새처럼 날지만 젖을 먹여 키웁니다."
  }
]
```

### 8-2. `state.json`
현재 프로그램 상태를 저장하는 파일입니다.  
퀴즈 추가/삭제, 최고 점수, 점수 기록 히스토리가 모두 이 파일에 반영됩니다.

예시 구조:

```json
{
  "quizzes": [
    {
      "question": "박쥐는 어떤 분류에 속할까요?",
      "choices": ["조류", "파충류", "포유류", "양서류"],
      "answer": 3,
      "hint": "새처럼 날지만 젖을 먹여 키웁니다."
    }
  ],
  "best_score": 80,
  "score_history": [
    {
      "played_at": "2026-04-12 20:30:00",
      "question_count": 5,
      "correct_count": 4,
      "hint_used_count": 1,
      "score": 75
    }
  ]
}
```

## 9. 예외 처리

프로그램이 비정상 종료되지 않도록 다음 상황을 처리했습니다.

### 입력 관련 예외 처리
- 빈 입력
- 숫자가 아닌 입력

- `EOFError`

### 파일 관련 예외 처리

- `state.json` 파일 없음
- JSON 형식 손상
- 잘못된 퀴즈 데이터 구조
- 파일 읽기/쓰기 오류

오류가 발생할 경우 가능한 범위에서 기본값으로 복구하거나, 안전하게 종료하도록 구현했습니다.

## 10. 점수 계산 방식

점수는 아래 방식으로 계산합니다.

- 기본 점수 = `(정답 수 / 전체 문제 수) * 100`
- 힌트 1회 사용 시 5점 차감
- 최종 점수는 0점 미만으로 내려가지 않음

예를 들어 5문제 중 4문제를 맞혀 기본 점수가 80점이고, 힌트를 1회 사용했다면 최종 점수는 75점입니다.
