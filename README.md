# 미션: Git과 함께하는 Python 첫 발자국

## 1. 프로젝트 개요

이 프로젝트는 Python으로 만든 콘솔 기반 퀴즈 게임입니다.
사용자는 메뉴를 통해 퀴즈를 풀고, 새로운 퀴즈를 추가하고, 등록된 퀴즈 목록을 확인하고, 최고 점수를 확인할 수 있습니다.
또한 프로그램을 종료한 뒤 다시 실행해도 퀴즈 데이터와 점수 기록이 유지되도록 `state.json` 파일을 사용해 데이터를 저장합니다.

이 프로젝트는 단순히 동작하는 프로그램을 만드는 데 그치지 않고, 다음 내용을 함께 연습하는 것을 목표로 합니다.

* Python 기본 문법을 이용한 콘솔 프로그램 구현
* 클래스 기반 설계와 역할 분리
* JSON 파일을 통한 데이터 저장과 복구
* Git/GitHub를 활용한 기능 단위 개발과 버전 관리

추가로 보너스 과제도 함께 반영하여, 기본 기능 외에 문제 순서 랜덤 출제, 문제 수 선택, 힌트 기능, 퀴즈 삭제 기능, 점수 기록 히스토리 저장까지 구현하는 것을 목표로 합니다.

## 2. 퀴즈 주제 선정 이유

이 프로젝트의 퀴즈 주제는 **[생물]** 입니다.

퀴즈 주제는 단순히 문제를 만들기 쉬운 분야보다, 객체의 속성과 역할을 설계하기 좋은 방향으로 정하고 싶었다. 생물 주제는 동물, 인간, 서식 환경, 분류 체계, 특징 등 다양한 정보를 문제 데이터에 반영할 수 있어 객체지향 구조를 연습하기에 적합하다고 보았습니다.

## 5. 프로그램 동작 방식

### 5-1. 퀴즈 풀기

사용자는 저장된 퀴즈 중 일부 또는 전체를 선택해 풀 수 있습니다.
퀴즈를 시작하면 문제와 4개의 선택지가 출력되며, 사용자는 정답 번호를 입력합니다.
각 문제마다 정답 여부를 확인할 수 있고, 모든 문제를 푼 뒤 최종 점수가 출력됩니다.

추가 기능으로 다음 요소를 포함할 예정입니다.

* 문제 순서 랜덤 섞기
* 퀴즈 시작 전에 몇 문제를 풀지 선택하기
* 힌트 보기
* 힌트 사용 시 점수 차감

### 5-2. 퀴즈 추가

사용자는 직접 새로운 문제를 등록할 수 있습니다.
문제, 선택지 4개, 정답 번호를 입력하면 새 퀴즈가 목록에 추가되고 파일에도 저장됩니다.

### 5-3. 퀴즈 목록

현재 저장된 퀴즈 목록을 확인할 수 있습니다.
문제 번호와 문제 내용이 출력되며, 퀴즈가 하나도 없는 경우에는 안내 메시지를 출력합니다.

### 5-4. 점수 확인

최고 점수와 점수 기록 히스토리를 확인할 수 있습니다.
기본 요구사항에서는 최고 점수 확인이 핵심이며, 보너스 기능으로 날짜/시간, 푼 문제 수, 점수 기록까지 함께 저장합니다.

### 5-5. 퀴즈 삭제

등록된 퀴즈 중 원하는 번호를 선택해 삭제할 수 있습니다.
삭제 후에는 파일에도 반영되도록 구현할 예정입니다.

## 6. 코드 구조

이 프로젝트는 최소 2개 이상의 클래스를 사용해 역할을 분리하여 구현합니다.

### `Quiz`

개별 퀴즈 1개를 표현하는 클래스입니다.

#### 주요 속성

* `question`: 문제 내용
* `choices`: 선택지 4개
* `answer`: 정답 번호
* `hint`: 힌트 내용(보너스 기능)

#### 주요 역할

* 문제 정보 보관
* 문제 출력
* 정답 확인
* 파일 저장용 데이터 변환

### `QuizGame`

게임 전체 흐름을 관리하는 클래스입니다.

#### 주요 속성

* `quizzes`: 전체 퀴즈 목록
* `best_score`: 최고 점수
* `score_history`: 점수 기록 히스토리
* `state_file`: 저장 파일 경로

#### 주요 역할

* 메뉴 출력
* 사용자 입력 처리
* 퀴즈 시작 및 진행
* 퀴즈 추가
* 퀴즈 목록 출력
* 점수 확인
* 퀴즈 삭제
* 파일 저장/불러오기
* 예외 처리 및 종료 처리

필요하다면 구현 과정에서 책임 분리를 더 명확하게 하기 위해 보조 클래스를 추가할 수 있습니다.
예를 들어 저장/불러오기를 담당하는 별도 클래스를 둘 수도 있지만, 현재 과제 기준에서는 `Quiz`와 `QuizGame` 두 클래스를 중심으로 설계할 예정입니다.

예상 파일 구조는 다음과 같습니다.

```bash
project/

   └─ screenshots/

      ├─ score.png
      └─ git_log.png
```

### 파일 설명

* `main.py`
  프로그램 실행 파일입니다.

* `quiz.py`
  `Quiz` 클래스를 정의하는 파일입니다.

  `QuizGame` 클래스를 정의하고 전체 게임 로직을 담당하는 파일입니다.

* `state.json`
  퀴즈 데이터, 최고 점수, 점수 히스토리를 저장하는 파일입니다.

* `README.md`
  프로젝트 설명 문서입니다.

* `.gitignore`
  Git으로 추적하지 않을 파일을 설정합니다.

* `docs/screenshots/`
  제출용 실행 화면 및 Git 로그 스크린샷을 저장할 폴더입니다.

## 8. 데이터 파일 설명

프로그램 데이터는 프로젝트 루트의 `state.json` 파일에 저장합니다.
파일 인코딩은 UTF-8을 기준으로 사용합니다.

### 저장 데이터

* 등록된 퀴즈 목록
* 최고 점수
* 점수 기록 히스토리

### 예상 데이터 구조

```json
{
  "quizzes": [
    {
      "question": "Python의 창시자는?",
      "choices": ["Guido", "Linus", "Bjarne", "James"],
      "answer": 1,
      "hint": "이름이 Guido로 시작합니다."
    }
  ],
  "best_score": 80,
  "score_history": [
    {
      "played_at": "2026-04-09 20:00:00",
      "question_count": 5,
      "score": 80
    }
  ]
}
```

### 필드 설명

* `quizzes`
  등록된 퀴즈 목록입니다.

* `best_score`
  현재까지 기록된 최고 점수입니다.

* `score_history`
  게임 플레이 기록 목록입니다.
  보너스 기능으로 날짜/시간, 푼 문제 수, 점수를 저장합니다.

## 9. 예외 처리 계획

프로그램이 비정상 종료되지 않도록 다음 상황을 처리할 예정입니다.

### 공통 입력 예외 처리

* 빈 입력
* 숫자가 아닌 입력
* 앞뒤 공백이 포함된 입력
* 허용 범위를 벗어난 숫자 입력

### 실행 중 예외 처리

* `KeyboardInterrupt` 발생 시 안내 메시지 출력 후 저장 가능한 데이터 저장
* `EOFError` 발생 시 안전하게 종료

### 파일 예외 처리

* `state.json` 파일이 없는 경우 기본 퀴즈 데이터로 시작
* `state.json` 파일이 손상된 경우 안내 메시지 출력 후 기본 데이터로 복구
* 파일 읽기/쓰기 중 오류가 발생해도 가능한 범위에서 종료 처리
